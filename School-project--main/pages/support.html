About

Folio is a small four-room application: an editor for plain-text files, a notebook for side notes, an AI summarizer, and an AI quiz generator. Open a .txt file, work on it, and use the AI tools to think with it.

Everything runs in your browser. There's no account and no server.

How it works

  • Editor — open any .txt (or .md) file with the Open button. Edit. Save back to disk with Save .txt. Your work auto-saves to this browser too, so you won't lose anything if you close the tab by accident.
  • Notes — a separate scratchpad for thoughts that don't belong in your main document. Each note has a Summarize button.
  • Quiz — paste any text or pull in the current document; the AI writes a 6–8 question multiple-choice quiz on it. Useful for studying.
  • Summarize — same idea, but for distilling. Choose Short, Medium, or Long.

The Summarize → and Make a quiz → buttons under the editor jump straight to those pages with your current document already loaded.

Keyboard shortcuts

  • Ctrl/Cmd + S — save the current document as a .txt file
  • Ctrl/Cmd + O — open a file
  • Tab — insert two spaces (without losing focus)

Privacy

Your API key, document, and notes live in your browser's localStorage on this device. They are not sent anywhere except, in the case of AI requests, directly to Groq's API.

Clearing your browser's site data will erase everything — there is no cloud backup. Save important documents to disk with the Save .txt button.

Troubleshooting

  • "401 Unauthorized" — the key was rejected. Re-paste it carefully (no spaces, the whole gsk_… string).
  • "429 Too Many Requests" — you've hit Groq's free-tier rate limit. Wait a minute and try again.
  • "Could not parse quiz" — the model returned something malformed. Click Generate quiz again; it usually works on the second try.
  • Document looks weird after opening — the file may not be UTF-8 plain text. Folio is built for .txt and .md files; binary formats like .docx or .pdf won't open correctly here.
  • Hosted via file:// — should work for everything since requests go to Groq directly. If you hit issues, run a tiny local server (python -m http.server) from the project folder.
